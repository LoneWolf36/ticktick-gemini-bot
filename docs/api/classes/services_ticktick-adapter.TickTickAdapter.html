    Class TickTickAdapter

    TickTick Adapter - Narrow interface for all TickTick REST API interactions. Wraps TickTickClient with validation, error classification, and structured logging.

    Example

    const adapter = new TickTickAdapter(new TickTickClient(credentials));
    await adapter.createTask({ title: 'My Task', projectId: 'abc123...' });

          Task description/notes

        • OptionaldueDate?: string

          ISO 8601 date string (e.g., '2025-04-01T09:00:00.000Z')

        • Optionalpriority?: number

          Priority level: 0=none, 1=low, 3=medium, 5=high

        • OptionalprojectId?: string

        • OptionalrepeatFlag?: string

        • title: string

          Task title (required, non-empty)

      Created task object from TickTick API

      Throws

      Classified error with code 'VALIDATION_ERROR' for invalid fields, or API error codes

      Parameters

      • taskId: string

        24-char hex task ID

      • normalizedAction: {
            content?: string;
            dueDate?: string;
            mergeContent?: boolean;
            originalProjectId?: string;
            priority?: number;
            projectId?: string;
            repeatFlag?: string;
            title?: string;
        }

        Normalized action with update fields

        • Optionalcontent?: string

          Content to merge or replace

        • OptionaldueDate?: string

          New due date

        • OptionalmergeContent?: boolean

          If false, replace content entirely; if true/undefined, merge with existing

        • OptionaloriginalProjectId?: string

          Original project ID (for cross-project moves)

        • Optionalpriority?: number

          New priority

        • OptionalprojectId?: string

          Target project ID (move task if different from original)

        • OptionalrepeatFlag?: string

          New recurrence rule

        • Optionaltitle?: string

          New title (replaces existing)

      • Optionaloptions: { verifyAfterWrite?: boolean } = {}

        Adapter options

        • OptionalverifyAfterWrite?: boolean

          Fetch task after update to verify fields

      Returns Promise<Object>

      Updated task object from TickTick API with verified and verificationNote

      Throws

      Classified error with code if API call fails or validation fails

      Example

      // Merge content (default behavior)
      await adapter.updateTask('task123...', { content: 'Additional note' });

      Example

      // Replace content entirely
      await adapter.updateTask('task123...', { content: 'New content only', mergeContent: false });
